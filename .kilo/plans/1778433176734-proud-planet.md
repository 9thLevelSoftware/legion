# Plan: Fix Kilo CLI Installer Registration & Publish to npm

## Problem Statement
The user runs the installer via `npx @9thlevelsoftware/legion --kilo`, but it falls back to interactive runtime selection showing 9 options without Kilo CLI. The screenshot shows an older runtime list (e.g., "Amazon Q Developer" as option 6), which does not match the current `main` branch.

## Root Cause Analysis
1. **Primary:** `npx` pulls from the npm registry, not the local repository. The published npm package is stale and does not contain the Kilo CLI support that was added to the local repo in v7.6.0. The local source already works — bash verification confirms `resolveRuntimeKey('--kilo')` returns `'kilo'` and all installer tests pass.
2. **Secondary (pre-publish cleanup):** `bin/runtime-metadata.js` contains a **duplicate `opencode` object literal** (lines 354–409 and 410–465). The second entry overwrites the first harmlessly, but it is a copy-paste regression that should be removed before publishing.

## Proposed Changes

### 1. `bin/runtime-metadata.js` — Remove duplicate `opencode` block
- Delete the redundant `opencode` entry at lines 410–465 (keep the first one at 354–409).
- This is a no-op behaviorally but fixes a real structural bug before publish.

### 2. `checksums.sha256` — Regenerate
- After editing `runtime-metadata.js`, run `node scripts/generate-checksums.js` to update the integrity manifest so `--verify` installs remain valid.

### 3. `package.json` — Version bump (if needed)
- Currently at `7.6.0`. If `7.6.0` has already been published to npm without Kilo support, bump to `7.6.1`. If `7.6.0` has never been published, keep it.
- Also update `CHANGELOG.md` date if bumping.

### 4. Test & Validate
- Run `node --test` (or `npm test`) to confirm all 1289 tests still pass, especially:
  - `runtime metadata has complete evidence-backed contracts`
  - `installer local mode installs runtime-native artifacts for every supported runtime`
  - `installer global mode installs runtime-native artifacts for every globally supported runtime`

### 5. Publish to npm
- Run `npm publish` (or `npm publish --access public` if scoped) so `npx` resolves the latest version with Kilo support.

## Files to Modify
- `bin/runtime-metadata.js` (remove duplicate `opencode` block)
- `checksums.sha256` (regenerate)
- `package.json` (version bump if 7.6.0 already published)
- `CHANGELOG.md` (version/date sync if bumping)

## Expected Outcome
- Cleaner runtime registry with no duplicate keys.
- Integrity hashes synchronized with source.
- Full test suite passes.
- Published npm package contains Kilo CLI support.
