## Key Patterns Identified

### Pattern 1: Rigid Performance/Quality Thresholds
Multiple testing agents (api-tester, performance-benchmarker) specify hard numeric thresholds (200ms, 10x load, 0.1% error) without escape valves for legitimate exceptions. Recommend converting to "defaults with override" pattern.

### Pattern 2: Absolute Language in Critical Rules
Most agents use "never", "always", "must" in their Critical Rules sections without exception paths. This is most problematic in:
- feedback-synthesizer (7 "Never" statements)
- qa-verification-specialist ("ALWAYS", "no exceptions")

### Pattern 3: Schema Non-Conformance
product-technical-writer.md includes a `tools` field not in the CLAUDE.md schema specification.

### Pattern 4: Positive Pattern - Workflow Optimizer Differentiation
testing-workflow-optimizer.md provides an excellent template for scope differentiation with explicit routing to related agents. This pattern should be replicated.
