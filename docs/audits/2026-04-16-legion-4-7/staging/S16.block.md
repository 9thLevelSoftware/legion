### Key Themes

1. **Missing lint_commands (CAT-9):** 8 of 9 CLI adapters lack the `lint_commands` field despite CLAUDE.md referencing it as conformance metadata. The base ADAPTER.md spec also omits it.

2. **Read-only enforcement gaps (CAT-6/CAT-3):** Most adapters rely on instruction-based read-only mode without platform enforcement. Gemini CLI, Codex CLI, Kiro CLI, and Cursor all document this limitation without adequate fallbacks.

3. **Model tier defaults undefined (CAT-6):** Aider, Kiro CLI, OpenCode, and Windsurf specify model tiers as "user-configured" without defaults, leaving runtime behavior undefined when users have not configured models.

4. **Parallel execution risks (CAT-3):** Gemini CLI's P1 finding highlights file conflict risks in parallel execution with no enforcement mechanism. Cursor's polling lacks timeout handling.

5. **Silent degradation (CAT-3):** Kiro CLI subagent tool limitations degrade silently. Aider's verification commands cannot be run. Windsurf's Turbo Mode may bypass user gates.

### Cross-Cutting Issues

- All adapters should implement the `lint_commands` field or the ADAPTER.md spec should remove it from conformance requirements
- Consider a standard "read-only verification" pattern that all adapters can reference
- Model tier fallback behavior should be standardized across adapters
