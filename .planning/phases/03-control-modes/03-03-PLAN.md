```markdown
# Control Modes

## Overview

Control modes adjust how strictly Legion's authority matrix rules are enforced during agent execution. Set the mode in `settings.json` to match your workflow's trust level.

## Quick Reference

| Mode | When to Use | Authority | Filtering | Approval | File Lock | Read-Only |
|------|------------|-----------|-----------|----------|-----------|-----------|
| autonomous | Rapid prototyping, solo dev | Off | Off | Off | Off | Off |
| guarded | Team workflows (default) | On | On | On | Off | Off |
| advisory | Code review, exploration | Off | Off | Off | Off | On |
| surgical | Hotfixes, compliance | On | On | On | On | Off |

## Mode Details

### autonomous
{2-3 paragraphs: what it does, when to use, risks to be aware of}

### guarded
{2-3 paragraphs: default mode, what it enforces, why it's the safe default}

### advisory
{2-3 paragraphs: read-only behavior, auto-commit suppression, use cases}
IMPORTANT: Note that advisory mode relies on prompt-level instructions, not hard enforcement. Agents are instructed not to modify files, but there is no filesystem-level sandbox preventing writes. For CLIs that support --dry-run or equivalent, the adapter should map advisory to that flag.

### surgical
{2-3 paragraphs: file-scope restriction, maximum safety, compliance scenarios}
Note: file-scope enforcement is two-layered: (1) preventive — prompt injection tells agents to stay within files_modified, (2) detective — post-execution check verifies only listed files were touched.

## Configuration

### Setting the Mode
Show settings.json example with each mode value.

### Mode Profiles
Explain control-modes.yaml structure. Profiles are boolean flag sets consumed by authority-enforcer.

### Fallback Behavior
What happens if control-modes.yaml is missing (guarded defaults). What happens if control_mode is omitted from settings.json (guarded default via Settings Resolution Protocol).

## How Modes Affect Commands

| Command | autonomous | guarded | advisory | surgical |
|---------|-----------|---------|----------|----------|
| /legion:build | No constraints | Authority enforced | Suggest only | Strict file scope |
| /legion:review | Unfiltered findings | Domain-filtered | Unfiltered findings | Domain-filtered |
| /legion:quick | No constraints | Authority enforced | Suggest only | Strict file scope |

## Known Limitations

- Advisory mode is a soft control — prompt instructions, not a filesystem sandbox
- Surgical mode's post-execution check detects but cannot prevent unauthorized file writes
- No partial overrides — presets are all-or-nothing (custom mode composition is a future enhancement)

## Choosing the Right Mode

Decision guide in text:
- Working alone on a prototype? → autonomous
- Working with a team on a real project? → guarded
- Just exploring or reviewing code? → advisory
- Touching production or compliance-critical code? → surgical
```
