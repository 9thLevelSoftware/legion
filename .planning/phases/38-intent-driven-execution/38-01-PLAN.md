```yaml
intents:
  harden:
    description: "Security audit with Testing + Security divisions"

        - testing-reality-checker

      secondary:
        - testing-api-tester

  document:
    description: "Generate documentation without implementation"
    agents:
      primary:
        - product-technical-writer
      secondary:
        - engineering-frontend-developer  # For component docs
    filter:
      include_task_types: ["documentation", "readme", "api-docs", "guides"]
      exclude_task_types: ["implementation", "code", "testing"]
    mode: "filter_plans"
    
  skip-frontend:
    description: "Exclude frontend/UI tasks"
    filter:
      exclude_agents:
        - engineering-frontend-developer
        - design-ui-designer
      exclude_file_patterns:
        - "src/frontend/**"
        - "*.tsx"
        - "*.jsx"
        - "*.css"
        - "*.scss"
        - "*.vue"
      exclude_task_types: ["ui-components", "frontend-architecture", "styling"]
    mode: "filter_plans"
    
  skip-backend:
    description: "Exclude backend/API tasks"
    filter:
      exclude_agents:
        - engineering-backend-developer
        - engineering-api-developer
      exclude_file_patterns:
        - "src/api/**"
        - "src/backend/**"
        - "*.route.ts"
        - "prisma/schema.prisma"
      exclude_task_types: ["api-endpoints", "database-schema", "backend-logic"]
    mode: "filter_plans"
    
  security-only:
    description: "Security-only review audit"
    agents:
      primary:
        - engineering-security-engineer
      secondary:
        - testing-api-tester  # For API security testing
    domains: ["security", "owasp", "stride", "authentication", "authorization"]
    mode: "filter_review"

task_types:
  # Taxonomy for filtering
  documentation: ["readme", "api-docs", "guides", "tutorials", "comments"]
  implementation: ["feature", "bug-fix", "refactor", "optimization"]
  testing: ["unit-test", "integration-test", "e2e-test", "security-test"]
  ui-components: ["react", "vue", "angular", "svelte", "web-components"]
  frontend-architecture: ["routing", "state-management", "build-config"]
  api-endpoints: ["rest", "graphql", "grpc", "webhook"]
  database-schema: ["migration", "model", "relation", "index"]

validation:
  mutual_exclusion:
    - flags: ["harden", "document"]
      error: "Cannot use --just-harden and --just-document together. Choose one intent."
    - flags: ["document", "skip-frontend"]
      error: "--just-document already excludes implementation; --skip-frontend is redundant."
    - flags: ["skip-frontend", "skip-backend"]
      error: "Cannot skip both frontend and backend — nothing to build."
      
  requires_command:
    - flag: "harden"
      commands: ["build"]
      error: "--just-harden is only valid for /legion:build"
    - flag: "document"
      commands: ["build"]
      error: "--just-document is only valid for /legion:build"
    - flag: "skip-frontend"
      commands: ["build"]
      error: "--skip-frontend is only valid for /legion:build"
    - flag: "skip-backend"
      commands: ["build"]
      error: "--skip-backend is only valid for /legion:build"
    - flag: "security-only"
      commands: ["review"]
      error: "--just-security is only valid for /legion:review"
      
  allowed_combinations:
    - flags: ["skip-frontend"]
      allows: ["skip-backend"]  # Can combine skips
    - flags: ["harden"]
      allows: []  # harden is exclusive
```

**Skill Structure:**

1. **Intent Flag Parsing** (Section 1)
   - parseIntentFlags(arguments): Extract --just-* and --skip-* flags
   - Return normalized intent objects with flag names and values
   - Handle flag combinations and edge cases

2. **Validation Engine** (Section 2) 
   - validateFlagCombination(intents, command): Apply validation rules
   - Check mutual_exclusion rules
   - Check requires_command constraints
   - Check allowed_combinations
   - Return { valid: boolean, errors: string[], suggestions: string[] }

3. **Team Template Resolution** (Section 3)
   - loadIntentTeams(): Load and cache intent-teams.yaml
   - resolveTeamTemplate(intent): Get agents, domains, mode for intent
   - resolveFilterCriteria(intent): Get filter predicates for filter_plans mode
   - mapDomainsToAgents(domains): Cross-reference authority matrix

4. **Execution Mode Detection** (Section 4)
   - detectExecutionMode(intent): Return "ad_hoc" | "filter_plans" | "filter_review"
   - getExecutionInstructions(mode): Return specific steps per mode

5. **Filter Predicates** (Section 5)
   - createAgentFilter(excludeAgents): Predicate function
   - createFileFilter(patterns): Glob-based predicate
   - createTaskFilter(includeTypes, excludeTypes): Content-based predicate
   - combineFilters(filters): AND logic for multiple predicates

6. **Integration Guide** (Section 6)
   - Usage in commands/build.md
   - Usage in commands/review.md
   - Error handling patterns
