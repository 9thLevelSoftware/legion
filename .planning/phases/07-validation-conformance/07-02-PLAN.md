2. **Command → Agent references**: If any command .md body contains an agent ID pattern (like `engineering-senior-developer` or text matching known agent IDs from CATALOG.md), verify the agent .md file exists at `agents/{agent-id}.md`.

3. **CATALOG.md → Agent file references**: Read `skills/agent-registry/CATALOG.md`. Extract all agent IDs from the catalog table (the ID column). For each ID, verify `agents/{agent-id}.md` exists.

4. **Agent file → CATALOG.md back-reference**: List all `agents/*.md` files. Extract the agent ID from the filename (strip `.md`). Verify each ID appears in the CATALOG.md catalog table.

Pattern for reading files:
```javascript

const COMMANDS_DIR = path.join(ROOT, 'commands');
const SKILLS_DIR = path.join(ROOT, 'skills');
```

For extracting execution_context, use regex to find content between `<execution_context>` and `</execution_context>` tags, then split by newlines and trim.

For extracting agent IDs from CATALOG.md, parse the table rows looking for `| {id} |` patterns in the Agent Catalog section.

<task type="auto">
  <name>Task 2: Create lint-commands test</name>
  <files>tests/lint-commands.test.js</files>
  <action>
Create `tests/lint-commands.test.js`. This test lints all command .md files for structural correctness.

Use `node:test` and `node:assert/strict`. No external dependencies.

The test must validate for every file in `commands/*.md`:

1. **Required YAML frontmatter fields**: Every command must have:
   - `name`: string, must match `legion:{command-name}` pattern
   - `description`: non-empty string
   - `argument-hint`: string (can be empty but field must exist)
   - `allowed-tools`: array of tool names

2. **Required XML sections**: Every command body must contain:
   - `<objective>` ... `</objective>` block
   - `<context>` ... `</context>` block
   - `<process>` ... `</process>` block

3. **No orphan closing tags**: No `</objective>`, `</context>`, or `</process>` without a matching opening tag.

4. **Execution context paths exist** (if `<execution_context>` is present): Each skill path referenced must exist on disk. This overlaps with cross-reference-validation but lint-commands catches it at the structural level per-file.

5. **Name consistency**: The `name` frontmatter field's command portion (after `legion:`) should match the filename (without `.md`). E.g., `plan.md` should have `name: legion:plan`.

Pattern for YAML frontmatter parsing: use the same regex approach as existing tests (find content between `---` delimiters, parse key-value pairs).
