**Section 1: Authority Loading**
Document how to load and parse the authority matrix:
```
Step 1: Load authority matrix
  - Read .planning/config/authority-matrix.yaml
  - Parse into memory: agent_id → exclusive_domains[] mapping
  - Build reverse index: domain → primary_agent_id

Step 2: Validate matrix integrity
  - Check all agent IDs exist in agent-registry
  - Check no domain is assigned to more than one agent as "exclusive"
  - Flag overlapping domains for conflict resolution
```

**Section 2: Boundary Validation**
Document how to check if an agent is within their authority:
```
Input: agent_id, topic, active_agents[]
Output: { authorized: boolean, domain_owner: string | null }

Algorithm:
1. Look up agent_id's exclusive_domains from matrix
2. If topic matches any exclusive_domain → authorized = true
3. If topic matches another agent's exclusive_domain:
   - If that agent is in active_agents[] → authorized = false, domain_owner = that_agent
   - Else → authorized = true (domain owner not present)
4. If topic matches no exclusive_domain → authorized = true (general domain)
```

**Section 3: Prompt Injection**
Document how to inject authority constraints into agent prompts:
```
Function: injectAuthorityConstraints(agent_id, base_prompt, active_agents[])

Steps:
1. Load agent's exclusive_domains from matrix
2. For each domain in exclusive_domains:
   - Add to prompt: "You have exclusive authority over {domain}. When active, other agents defer to you on this topic."
3. For each active_agent in active_agents (excluding self):
   - Load their exclusive_domains
   - Add to prompt: "{agent_id} has exclusive authority over {domains}. Do not critique or override their findings in these domains."
4. Return enhanced prompt
```

**Section 4: Finding Filtering**
Document how to filter findings during review synthesis:
```
Function: filterFindings(findings[], active_agents[])

Steps:
1. Build domain ownership map from active_agents
2. For each finding:
   - Identify the finding's domain from criterion or content
   - If domain has an owner in active_agents AND finding.reviewer != owner:
     - Discard finding with note: "Out-of-domain critique filtered — {owner} is domain authority"
   - Else: keep finding
3. Return filtered findings

Note: Domain detection uses keyword matching against authority matrix domains
```
