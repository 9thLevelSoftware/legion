### Task 1: Add Agent Frontmatter Parser
**Action**: Add a function to `recommendation-engine.js` that reads enriched metadata from agent .md frontmatter files.

1. Add `parseAgentMetadata()` function that:
   - Reads all `.md` files from the agents directory
   - Extracts YAML frontmatter between `---` fences
   - Parses `languages`, `frameworks`, `artifact_types`, `review_strengths` arrays using regex (same approach as Phase 5 contract tests -- no YAML parser dependency)
   - Returns a map of `agent-id` -> `{ languages, frameworks, artifact_types, review_strengths }`
2. Call `parseAgentMetadata()` in `recommendAgents()` and merge metadata into agent records from `parseCatalog()`
3. Handle graceful degradation: if an agent .md file is missing metadata fields, use empty arrays (don't break scoring)

**Verify**: `node -e "const {parseCatalog} = require('./scripts/recommendation-engine.js'); console.log('parse ok')"`
**Done when**: Agent metadata is loaded and merged with catalog data in the recommendation pipeline.

### Task 2: Add Metadata Scoring Layer
**Action**: Add a `metadataScore()` function and integrate it into the scoring pipeline.

1. Add `metadataScore(agent, concepts, promptLower)` function:
   - Exact match on `languages`: +3 points (prompt mentions "typescript" and agent has `typescript`)
   - Exact match on `frameworks`: +3 points (prompt mentions "react" and agent has `react`)
   - Exact match on `artifact_types`: +2 points (prompt mentions "tests" and agent has `tests`)
   - Exact match on `review_strengths`: +2 points (prompt mentions "security" review and agent has `security`)
   - Partial/substring match on any field: +1 point
2. Integrate into `recommendAgents()`:
   - Compute metadata score alongside semantic and heuristic scores
   - Add to total: `total = semantic + heuristic + metadataBoost + memoryBoost`
   - metadataBoost applies only when baseline (semantic + heuristic) > 0 (same gating as memory boost)
   - NOTE: The existing shortlist filter gates on `semantic > 0` (line 188). This means agents with semantic=0 but heuristic>0 only appear in the fallback path. Metadata boost does NOT change the shortlist filter -- it only affects ranking within the shortlist.
3. Include `metadataScore` in the recommendation output object alongside existing scores
4. Update `classifyConfidence()` to consider metadata score as a confidence booster:
   - If metadata >= 6, treat as equivalent to semantic >= 4 for confidence classification
5. Export `metadataScore` and `parseAgentMetadata` from `module.exports` alongside existing exports (needed for unit testing in Plan 06-03)

**Verify**: `node scripts/recommendation-engine.js "Build a React TypeScript frontend with accessibility"` -- should boost engineering-frontend-developer
**Done when**: Metadata scoring is integrated and affects agent ranking for technology-specific prompts.
