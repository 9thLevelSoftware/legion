### Task 1: Implement Archetype Boost Scoring
**Action**: Add archetype-weighted scoring to `recommendation-engine.js`.

1. Add `archetypeBoost(agent, taskType, archetypeScores)` function:
   - Input: agent record, the detected task_type from the prompt, and archetypeScores from memory recall
   - If no archetypeScores provided (memory not available): return 0
   - If agent has history for this task_type in archetypeScores:
     - Base boost = `successRate * 3.0` (mirrors memory scoring formula)
     - Volume modifier: `min(totalOutcomes / 5, 1.0)` (agents with more history get full boost)
     - Top agent bonus: +1.0 if agent is the `topAgent` for this task_type
     - archetype_boost = `base_boost * volume_modifier + top_agent_bonus`
     - Clamp to [0, 5] (same as memory boost ceiling)
   - If agent has no history for this task_type: return 0
2. Add task_type detection from prompt:
   - Map prompt concepts to CATALOG.md Task Type Index categories
   - Use SEMANTIC_MAP + new TASK_TYPE_MAP for mapping (e.g., "react" -> "web-development", "api" -> "api-development")
   - If no task_type detected: skip archetype boost entirely
3. Integrate into `recommendAgents()`:
   - Accept optional `archetypeScores` parameter (alongside existing `memoryScores`)
   - Compute archetype boost per agent: `archetypeBoost(agent, detectedTaskType, archetypeScores)`
   - Add to total: `total = semantic + heuristic + metadataBoost + memoryBoost + archetypeBoost`
   - Gate behind baseline: only apply when `semantic + heuristic > 0`
4. Include `archetypeBoost` in recommendation output object

**Verify**: `node scripts/recommendation-engine.js "Build a React TypeScript frontend"` -- check output includes archetype fields
**Done when**: Archetype boost scoring is integrated and affects agent ranking when historical task_type data is available.
