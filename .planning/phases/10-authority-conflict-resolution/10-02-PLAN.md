## Task 1: Define Agent Communication Protocol

**Action**: Create `.planning/config/agent-communication.yaml` defining message types, handoff conventions, SUMMARY.md export requirements, and agent discovery patterns.

1. Define the three communication message types for agent coordination:
   ```yaml
   message_types:
     handoff_context:
       purpose: "Pass structured output from one wave to the next"
       location: "SUMMARY.md → Handoff Context section"
       required_fields:
         - key_outputs      # list of files created/modified with purpose
         - decisions_made   # architectural or design decisions agents made
         - open_questions   # unresolved items the next wave should address
         - conventions_established  # patterns set that subsequent agents must follow
       optional_fields:
         - escalations_pending  # unresolved escalations from escalation-protocol
         - data_for_downstream  # structured data a downstream plan specifically needs

     dependency_request:
       purpose: "Signal that a plan needs specific output from another plan"
       location: "Plan frontmatter depends_on + context section"
       format: "Declared at planning time, not runtime. Plans list which prior SUMMARY.md sections they need."
       resolution: "Wave executor validates dependency SUMMARY.md exists before dispatching"

     status_update:
       purpose: "Communicate plan completion status within a wave"
       location: "Wave executor internal — not agent-facing"
       statuses: ["in_progress", "completed", "blocked", "escalated"]
   ```

2. Define SUMMARY.md export requirements — what every plan MUST include in its SUMMARY.md for downstream consumption:
   ```yaml
   summary_export_standard:
     required_sections:
       - "## Completed Tasks"          # what was done (task-level)
       - "## Files Modified"           # paths + what changed
       - "## Handoff Context"          # structured output for next wave
       - "## Verification Results"     # did verification commands pass
     conditional_sections:
       - "## Escalations"              # if any escalations occurred (from 10-01)
       - "## Decisions Made"           # if non-trivial decisions were needed
       - "## Open Questions"           # if unresolved items remain
   ```

3. Define agent discovery conventions — how agents learn about their execution context:
   ```yaml
   agent_discovery:
     at_spawn:
       - current_wave: "Which wave number this agent is in"
       - wave_agents: "List of other agents in the same wave (parallel peers)"
       - prior_wave_agents: "List of agents from prior waves (dependency sources)"
       - own_domains: "Domains from authority-matrix this agent owns"
     via_context:
       - prior_summaries: "SUMMARY.md files from depends_on plans"
       - phase_context: "CONTEXT.md with phase goal and plan structure"
       - escalations_pending: "Any unresolved escalations from prior waves"
   ```

4. Define cross-wave communication rules:
   - **Forward only**: Information flows from earlier waves to later waves via SUMMARY.md. No backward communication.
   - **No runtime messaging**: Agents don't send messages during execution. All communication is through artifacts (SUMMARY.md, files).
   - **Orchestrator mediation**: If Wave 2 agent discovers it needs clarification from Wave 1 output, it documents the gap as an `open_question` in its own SUMMARY.md. The orchestrator or human resolves it.
   - **Escalation inheritance**: Unresolved `pending` escalations from Wave 1 are passed to Wave 2 agents via handoff context, so they can work around or address them.

**Verify**: `cat .planning/config/agent-communication.yaml | head -30` shows message types
**Done when**: Communication protocol YAML defines message types, export requirements, discovery conventions, and cross-wave rules

## Task 2: Integrate Handoff Context into Wave Executor

**Action**: Update `skills/wave-executor/SKILL.md` to inject structured handoff context when dispatching Wave 2+ agents, and enforce SUMMARY.md export standards.

1. Add a new section — "Handoff Context Injection" (after escalation detection, before wave transition):
   - When dispatching a Wave 2+ plan:
     a. Read all SUMMARY.md files listed in the plan's `depends_on`
     b. Extract the "Handoff Context" section from each dependency SUMMARY.md
     c. Extract any pending escalations from the "Escalations" section
     d. Compile a handoff briefing:
        ```
        ## Handoff Context from Prior Wave

        ### From Plan {depends_on_id}: {plan_title}
        **Key outputs**: {key_outputs list}
        **Decisions made**: {decisions list}
        **Open questions**: {open_questions list}
        **Pending escalations**: {escalations with status: pending}
        ```
     e. Inject the handoff briefing into the agent's spawn prompt, after personality and before task instructions

2. Add agent discovery injection to all wave dispatches:
   - Before spawning each agent, compile discovery context:
     ```
     ## Execution Context
     - Wave: {N} of {total_waves}
     - Parallel peers: {list of agents in same wave, or "solo"}
     - Prior wave: {list of agents from prior wave, or "none (first wave)"}
     - Your domains: {domains from authority-matrix}
     ```
   - Inject discovery context after personality, before handoff context

3. Add SUMMARY.md export validation:
   - After each plan completes and SUMMARY.md is written:
     a. Validate required sections exist (Completed Tasks, Files Modified, Handoff Context, Verification Results)
     b. If missing: log a `warning` — "Plan {id} SUMMARY.md missing required section: {section}"
     c. Do NOT block execution for missing sections — downstream plans handle missing data gracefully
   - The validation is advisory, not blocking, to maintain Legion's "degrade gracefully" principle

4. Add wave transition handoff logging:
   - At each wave boundary (before starting next wave), log to build output:
     ```
     --- Wave {N} → Wave {N+1} Handoff ---
     Plans completed: {list}
     Handoff context available: {list of SUMMARY.md files}
     Pending escalations carried forward: {count}
     ```

**Verify**: `grep -c 'handoff' skills/wave-executor/SKILL.md` returns >= 10
**Done when**: Wave executor injects handoff context into Wave 2+ agents, provides agent discovery, validates SUMMARY.md exports, and logs wave transitions
